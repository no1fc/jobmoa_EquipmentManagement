## 계층별 역할

### 1. 클라이언트 계층

**Next.js (Web)**
- PC 환경에서 장비 관리 전체 기능 제공
- 대여 현황 대시보드 (실시간)
- 관리자 전용 기능 (통계, 사용자 관리)

**Flutter (Mobile)**
- 모바일 환경에서 장비 관리 + AI 인식
- Gemma 4 E2B: 카메라로 장비 촬영 → 온디바이스 AI가 장비 분류 추천
- 이미지는 외부로 전송되지 않음 (Privacy-First)
- Human-in-the-loop: AI 추천 → 사용자 확인/수정 → 등록

### 2. API 계층 (Spring Boot)

**모듈 구성:**
- **장비관리**: 장비 CRUD, 카테고리 관리, 이미지 메타데이터 저장
- **대여관리**: 대여/반납 처리, 대여 현황, 연체 관리
- **사용자**: 인증(JWT), 역할 관리, 프로필
- **알림서비스**: FCM 푸시, 이메일 발송, 알림 스케줄링

**인증 흐름:**
```
로그인 → JWT Access Token (30분) + Refresh Token (7일) 발급
모바일: Flutter Secure Storage에 저장
  웹: HttpOnly Cookie로 저장
API 요청마다 Authorization: Bearer <token> 헤더 첨부
```

### 3. 데이터 계층 (MSSQL)

**핵심 테이블:**
- `users` — 상담사/관리자 계정
- `assets` — 장비/가구 자산 (IT장비, 가구 통합)
- `asset_categories` — 장비 카테고리 (대/중/소분류)
- `rentals` — 대여/반납 이력
- `consumables` — 소모품 재고
- `notifications` — 알림 이력

## 데이터 흐름

### AI 장비 등록 (Mobile)
```
1. 상담사 → 카메라로 장비 촬영
2. Flutter → Gemma 4 E2B 로컬 추론 (3초 이내)
3. AI → 장비 카테고리 + 명칭 추천 표시
4. 상담사 → 확인/수정 후 추가 정보 입력
5. Flutter → Spring Boot API로 등록 요청 (이미지 메타데이터만)
6. Spring Boot → MSSQL 저장
```

### 대여/반납
```
1. 상담사 → 장비 선택 → 대여 신청
2. Spring Boot → 가용성 확인 → 대여 레코드 생성
3. 반납 예정일 도래 → 스케줄러 → FCM 푸시 + 이메일 발송
4. 상담사 → 반납 처리 → 상태 업데이트
```
