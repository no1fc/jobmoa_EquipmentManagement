# MVP 범위 및 Phase 로드맵

> PRD Phase 1 기준, 1인 개발

## Phase 1 — MVP (핵심 기능)

### P1 (필수 — 최우선)

| 기능 | 설명 | 플랫폼 |
|------|------|--------|
| 장비 목록 조회 | 등록된 전체 장비 목록 및 상태 확인 | PC/Mobile |
| 장비 수동 등록 | 장비명, 분류, 시리얼번호, 구매일자, 상태 수기 입력 | PC/Mobile |
| **AI 장비 등록** | **카메라 촬영 → 온디바이스 AI 분류 → Human-in-the-loop 확인** | **Mobile** |
| 장비 상세/수정 | 장비 상세 정보 조회 및 수정 | PC/Mobile |
| 대여 현황 대시보드 | 대여 중인 장비, 반납 예정일, 연체 여부 | PC/Mobile |
| 대여/반납 처리 | 대여 신청, 반납 확인, 상태 업데이트 | PC/Mobile |
| 사용자 인증 | JWT 로그인/로그아웃, 역할 기반 접근제어 | PC/Mobile |
| 반납 알림 | 반납 예정일 도래 시 인앱 푸시 + 이메일 | PC/Mobile |

### P2 (후순위 — MVP 이후)

| 기능 | 설명 | 플랫폼 |
|------|------|--------|
| 바코드/QR 스캔 | 장비 바코드/QR 스캔으로 대여/반납 즉시 처리 | Mobile |
| 소모품 수량 관리 | A4용지, 토너 등 소모품 입출고 및 잔여 수량 관리 | PC/Mobile |

## Phase 3 — 확장

| 기능 | 설명 |
|------|------|
| 다중 객체 인식 | 한 번의 촬영으로 공간 내 여러 장비 일괄 인식 |
| 오프라인 동기화 | 네트워크 단절 시에도 데이터 저장 후 복구 시 동기화 |
| 통계 대시보드 | 자산 활용률, 대여 통계, 카테고리별 현황 |

## 제외 (기획서 고급 기능)

다음 기능은 기획서에 포함되어 있으나 MVP 및 근시일 내 개발 범위에서 제외:

- 6단계 자산 생애주기 관리 (전략수립 → 조달 → 배포 → 운영 → 업그레이드 → 폐기)
- CRDT 기반 오프라인 충돌 해결
- E2B 클라우드 샌드박스 에이전틱 액션
- MTTR/MTBF/ROI/TCO KPI 대시보드
- GPS 텔레매틱스 연동
- ERP/HR 시스템 연동
- MDM 엔드포인트 관리 연동

## 개발 순서 (권장)

```
1. 백엔드 기반 구축
   └─ Spring Boot 프로젝트 초기화
   └─ DB 스키마 생성 (MSSQL)
   └─ JWT 인증 구현
   └─ 장비 CRUD API

2. 웹 프론트엔드
   └─ Next.js 프로젝트 초기화
   └─ 로그인 페이지
   └─ 장비 관리 화면 (목록/등록/수정)
   └─ 대여 대시보드

3. 모바일 앱
   └─ Flutter 프로젝트 초기화
   └─ 로그인 + API 연동
   └─ 장비 관리 화면
   └─ Gemma 4 E2B AI 장비 등록

4. 알림 시스템
   └─ FCM 푸시 알림
   └─ 이메일 알림
   └─ 반납 예정 스케줄러
```
