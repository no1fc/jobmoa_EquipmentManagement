* **작성일:** 2026-04-27  
* **버전:** v1.0  
* **작성자:** 제품기획팀 (Product Manager)  
* **대상 독자:** 개발팀(Web/Mobile/AI), 디자인팀, 프로젝트 이해관계자, 경영진

## **1\. 프로젝트 개요 (Project Overview)**

### **1.1. 제품 비전 및 목표**

본 프로젝트는 국민취업지원제도 지점에서 근무하는 일반 상담사들이 복잡한 IT 및 사무용 장비를 직관적이고 효율적으로 관리할 수 있도록 돕는 \*\*'통합 장비/자산 관리 플랫폼'\*\*을 구축하는 것입니다. 모바일 환경에 특화된 온디바이스 AI(Gemma 4 E2B)를 도입하여, 장비에 대한 전문 지식이 없는 상담사도 사진 촬영 한 번으로 장비를 식별하고 등록할 수 있는 혁신적인 사용자 경험을 제공합니다.

### **1.2. 대상 사용자 (Target Audience)**

* **일반 상담사 (Primary User):** 지점에서 내담자 상담을 주 업무로 하며, IT 기기나 사무용 가구(5단장, 3단장 등)의 정확한 명칭이나 스펙에 대한 전문 지식이 부족한 실무자.  
* **지점 관리자 (Secondary User):** 지점 내 전체 자산의 현황, 대여 기록, 재고 및 소모품 잔여량을 파악하고 통제해야 하는 책임자.

## **2\. 문제 정의 및 해결 방안 (Problem & Solution)**

### **2.1. 현재의 문제점 (Pain Points)**

* **장비 식별의 어려움:** 일반 상담사들은 PC 모델명, 프린터기 종류, 가구 규격(5단장/3단장) 등을 정확히 알지 못해 수기 기록 시 오류가 잦음.  
* **비효율적인 이력 관리:** 장비의 대여, 반납, 소모품 보충 기록이 파편화되어 있어 추적이 어려움.  
* **보안 및 서버 비용 이슈:** 기존 클라우드 기반 AI 비전 API를 사용할 경우, 보안 규정상 지점 내부 사진을 외부 서버로 전송하는 데 제약이 따르며 지속적인 API 호출 비용이 발생함.

### **2.2. 해결 방안 (Proposed Solution)**

* **크로스 플랫폼 지원:** PC 웹과 모바일 앱 환경에서 동일한 수준의 장비 관리 및 대여 관리 기능을 제공하여 접근성 극대화.  
* **온디바이스 AI 비전 인식 탑재 (모바일):** 4-bit로 양자화된 'Gemma 4 E2B' 비전-언어 모델을 모바일 기기에 탑재. 상담사가 장비 사진을 찍으면 AI가 오프라인 환경에서도 즉시 장비 종류(PC, 핸드폰, 프린터기, 5단장, 소모품 등)를 자동 분류 및 추천.  
* **Human-in-the-loop 방식 적용:** AI가 분석한 결과를 바탕으로 사용자가 최종 확인/수정하여 시스템에 등록하도록 설계해 데이터 정확도 100% 확보.

## **4\. 상세 기능 명세 (Detailed Feature Specifications)**

### **4.1. 모바일 AI 장비 등록 프로세스 (핵심 사용자 여정)**

1. **사진 촬영:** 모바일 앱에서 'AI 장비 등록' 버튼 탭 \-\> 카메라 실행 및 장비 촬영.  
2. **온디바이스 추론:** 모바일 내부에 적재된 Gemma 4 E2B 4-bit 모델이 촬영된 이미지를 분석.  
3. **결과 제공:** AI가 장비의 대분류(예: 가구), 중분류(예:올문장) 및 소분류(예: 5단장), 추정되는 모델 특징을 화면에 출력.  
4. **검토 및 수정 (Human-in-the-loop):** 상담사는 AI가 추천한 정보가 맞는지 확인. 틀린 경우 드롭다운 메뉴를 통해 올바른 카테고리(예: 3단장)로 손쉽게 수정.  
5. **추가 정보 입력 & 등록:** 취득일자, 보관 위치(ex. 제 1상담실) 등 필수 정보를 추가 입력 후 최종 등록. 바코드/QR 자동 생성 및 매핑.

### **4.2. 대여 관리 기능 명세**

* **대여 생성:** 대상 장비 선택 \-\> 대여사유 작성 \-\> 대여 기간 설정 \-\> 대여 대상자(상담사 또는 내담자) 지정.  
* **알림 기능:** 반납 예정일 도래 시 PC 웹 대시보드 알림 및 모바일 푸시 알림 전송.  
* **이력 추적:** 특정 노트북(PC)이 과거 누구에게 언제 대여되었는지 Tracking.

## **5\. 시스템 아키텍처 및 기술 스택 (System Architecture)**

* **프론트엔드 (PC Web / Mobile):** \* React.js (Web) / React Native 또는 Flutter (Mobile)  
  * 반응형(Responsive) 웹 디자인을 통한 크로스 플랫폼 일관성 확보.  
* **백엔드 (서버 & DB):**  
  * Node.js / Spring Boot 기반 RESTful API.  
  * DB: PostgreSQL (자산 메타데이터, 대여 이력 등 텍스트 데이터 보관).  
* **AI 모듈 (Mobile On-Device):**  
  * 모델: **Gemma 4 E2B (Vision-Language Model)**  
  * 최적화: 모바일 구동을 위한 **4-bit 양자화(Quantization)** 적용 (GGUF 또는 TFLite 포맷 변환).  
  * 실행 환경: 모바일 기기의 NPU/GPU를 활용한 로컬 추론 엔진 (e.g., MediaPipe, MLC LLM).

## **6\. 제약 사항 및 위험 관리 (Constraints & Risk Management)**

* **모바일 하드웨어 스펙 한계 (Risk):** 오래된 스마트폰의 경우 온디바이스 모델(Gemma 4\) 구동 시 발열이나 앱 강제 종료가 발생할 수 있음.  
  * *Mitigation (대응방안):* 앱 실행 전 기기 스펙을 체크하여, 스펙이 부족한 기기에서는 AI 자동 인식 기능 대신 '수동 등록' 화면으로 자연스럽게 Fallback(전환) 하도록 처리.  
* **AI 인식 정확도 문제 (Risk):** 3단장과 5단장을 헷갈리거나 덮여있는 프린터기를 일반 박스로 오인할 수 있음.  
  * *Mitigation (대응방안):* 사용자 UI에 "AI의 분석 결과는 참고용이며, 최종 확인은 사용자가 직접 해주세요"라는 안내 문구 상시 노출. 수정이 매우 간편한 UI 구조 채택.

## **7\. 성공 지표 (Success Metrics \- KPIs)**

* **AI 등록 활용률:** 신규 등록되는 장비 중 AI 카메라 기능을 거쳐 등록된 비율 (목표: 70% 이상).  
* **장비 등록 소요 시간 단축:** 기존 수기 입력 시 평균 소요 시간 vs AI 기능 사용 시 평균 소요 시간 (목표: 기존 대비 50% 단축).  
* **활성 사용자 수 (DAU/WAU):** 시스템을 매일/매주 활용하여 자산 상태를 확인하고 대여/반납을 처리하는 상담사 비율.  
* **데이터 무결성:** 장비 등록 후 카테고리 오기입으로 인한 데이터 수정 발생률 (목표: 5% 미만).

## **8\. 향후 로드맵 (Future Roadmap)**

* **Phase 1 (MVP):** PC/모바일 기본 장비/대여 관리 기능 및 모바일 온디바이스 AI(단일 장비 인식) 적용 배포.  
* **Phase 2:** QR/바코드 자동 생성 기능과 라벨 프린터기 연동을 통한 실물 자산 트래킹 고도화.  
* **Phase 3:** 다중 객체 인식. 한 번의 사진 촬영으로 공간 내 여러 장비(책상, 모니터, PC, 3단장 등)를 일괄로 인식하고 등록 목록으로 나열해주는 기능 개발.
